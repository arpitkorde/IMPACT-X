## The Problem

Cricket didn't just change its format. It changed what it means to be valuable.

In Test cricket, a batsman who survived 300 balls was celebrated — endurance was the virtue. Then ODIs arrived and volume became the metric. Then T20 happened — and suddenly the only question that mattered was: *what did you do while you were there? Did it help the team win?*

The professional world is at the same inflection point.

For decades, human contribution at work has been measured in hours. We built entire systems — billing models, appraisals, office cultures — around the idea that time is a proxy for value.

Then AI arrived.

When one person with the right tools can do in 3 hours what once took a team 3 weeks — the hour stops meaning anything. But here's what nobody has solved yet: **when a human and an AI agent work together on a task, how do you measure what each contributed? And how do you measure the value of the combination?**

That's what IMPACT-X is built to answer.

## What IMPACT-X Measures

IMPACT-X is a three-layer open framework for measuring contribution in an AI-augmented world:

```
Layer 1 → Human IMPACT Score
Layer 2 → Agent IMPACT Score  
Layer 3 → Combined IMPACT Score (Human + AI Team)
```

**Layer 1** measures what the human contributed across six dimensions — Intelligence Leverage, Multiplier Effect, Problem Depth, Acceleration, Client Value, and Team Contribution.

**Layer 2** measures what the AI agent or tool contributed — Accuracy Rate, Task Compression, Problem Fit, Reliability Consistency, and Human Amplification Rate.

**Layer 3** measures what the human-AI team achieved together — including a Synergy Multiplier that captures whether the combination created more value than either could alone.

The framework is:
- **Context-first** — not all work is the same. A surgeon in theatre and a developer writing code are measured differently.
- **Living** — built to evolve as AI capabilities change, through Annual Zone Reviews and Trigger Event Protocols.
- **Fair** — includes an explicit Fairness Protocol ensuring access and training prerequisites before any measurement begins.
- **Open** — every dimension, formula, and benchmark is community-maintained and freely available.

## Quick Start

**For individuals:** Read [FRAMEWORK.md](FRAMEWORK.md) and use the [scoring tool](scoring-tool/index.html) to run your first IMPACT score.

**For team managers:** Start with the [Work Zone Mapping template](templates/work-zone-mapping.md) — a 2-hour workshop that maps your team's tasks into Preparation, Execution, and Integration zones before any scoring begins.

**For organisations:** Follow the [Pilot Checklist](templates/pilot-checklist.md) — a 6-week implementation guide that runs IMPACT alongside your existing metrics before replacing anything.

**For contributors:** Read [CONTRIBUTING.md](CONTRIBUTING.md) to understand how to add case studies, build vertical adaptations, improve the scoring formulas, or challenge the framework itself.

## Why Open Source

Frameworks without field testing are just philosophy.

IMPACT-X is open source because the only way to know whether this works — across industries, roles, cultures, and organisation sizes — is to let practitioners in every context test it, break it, improve it, and report back.

Every contribution makes the framework more honest. Every case study makes the benchmark more meaningful. Every challenged assumption makes the scoring more defensible.

This is how a framework becomes a standard.

## The North Star

IMPACT-X succeeds when it can make this claim with evidence:

> *Teams using the IMPACT-X framework show measurably better retention of high-contribution employees, faster project delivery, higher stakeholder satisfaction, and clearer understanding of which human-AI combinations create genuine synergy — compared to teams using hours-based measurement alone.*

Everything this community builds works toward generating the data to make that claim.
